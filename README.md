# nix-excalirender

Nix packaging for [`JonRC/excalirender`](https://github.com/JonRC/excalirender).

This package builds `excalirender` from source, installs Bun dependencies via [bun2nix](https://github.com/nix-community/bun2nix),
and rebuilds the native `canvas` addon inside Nix so the final package works on Linux and Darwin.

The repository is intentionally small:

- `flake.nix`: flake outputs and overlay wiring
- `package.nix`: the package definition
- `bun.nix`: Bun dependency metadata generated by `bun2nix`

## Usage

Run it once:

```bash
nix run github:sagikazarmark/nix-excalirender#excalirender -- --help
```

Add it to your dev shell in `flake.nix`:

```nix
{
  inputs = {
    nixpkgs.url = "github:NixOS/nixpkgs/nixpkgs-unstable";
    excalirender.url = "github:sagikazarmark/nix-excalirender";
    excalirender.inputs.nixpkgs.follows = "nixpkgs";
  };

  outputs = { self, nixpkgs, excalirender, ... }:
    let
      system = "aarch64-darwin"; # Replace this with your system
      pkgs = import nixpkgs { inherit system; };
    in
    {
      devShells.${system}.default = pkgs.mkShell {
        packages = [ excalirender.packages.${system}.excalirender ];
      };
    };
}
```

Or use the overlay:

```nix
{
  inputs = {
    nixpkgs.url = "github:NixOS/nixpkgs/nixpkgs-unstable";
    excalirender.url = "github:sagikazarmark/nix-excalirender";
    excalirender.inputs.nixpkgs.follows = "nixpkgs";
  };

  outputs = { self, nixpkgs, excalirender, ... }:
    let
      system = "aarch64-darwin"; # Replace this with your system
      pkgs = import nixpkgs {
        inherit system;
        overlays = [ excalirender.overlays.default ];
      };
    in
    {
      devShells.${system}.default = pkgs.mkShell {
        packages = [ pkgs.excalirender ];
      };
    };
}
```

## Upgrading

When upstream releases a new version:

1. Update `version` in `package.nix`.
2. Regenerate `bun.nix` from the new upstream `bun.lock`.
3. Update the source hash in `package.nix`.
4. Build and smoke-test the package.

Regenerate `bun.nix`:

```bash
version=1.10.5 # replace with the target release
tmpdir=$(mktemp -d)
curl -fsSL -o "$tmpdir/bun.lock" \
  "https://raw.githubusercontent.com/JonRC/excalirender/v${version}/bun.lock"
nix run github:nix-community/bun2nix -- \
  -l "$tmpdir/bun.lock" \
  -o bun.nix
```

Update the source hash:

1. Temporarily set `hash = lib.fakeHash;` in `package.nix`.
2. Run `nix build .`.
3. Copy the hash from the failure output back into `package.nix`.
4. Run `nix build .` again.

Smoke-test the updated package:

```bash
nix build .
./result/bin/excalirender --help
```

Optional end-to-end check:

```bash
version=1.10.5
tmpdir=$(mktemp -d)
curl -fsSL -o "$tmpdir/sample.excalidraw" \
  "https://raw.githubusercontent.com/JonRC/excalirender/v${version}/sample.excalidraw"
./result/bin/excalirender "$tmpdir/sample.excalidraw" -o "$tmpdir/out.png"
ls "$tmpdir/out.png"
```

**Notes**

- `bun.nix` is generated and should not be edited manually.
- The package explicitly rebuilds `canvas` from source because the Bun install step alone does not produce the native addon needed at runtime.
- If upstream changes native dependencies, `buildInputs` in `package.nix` may need updating as well.
